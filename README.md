# arrowbound

arrowbound 是一款面向桌面浏览器的横版 2D 射箭增量 Roguelite 游戏。

玩家需要观察弓的摆动角度，选择时机射出具有真实重力轨迹的箭矢。命中靶子的不同环数会掉落不同价值的金币；金币可以用于永久商店升级、解锁机器人、培养金币宠物并推进关卡。

## 当前状态

首版 MVP 阶段 0～11 已完成，阶段 12（UI、反馈与首版验收）收尾中。完整核心循环可在桌面浏览器游玩：

- 核心射箭手感：弓摆动、点击即射、重力抛物线、连续靶面命中判定
- 环数判定：1～10 环与脱靶，十环占比含商店/祝福加成并受 30% 上限约束
- 金币闭环：按来源与倍率固定价值、分档掉落、鼠标悬停经统一入口拾取入账
- 关卡推进：10 关、原子扣费通关解锁、自由返回、切关自动收金币与淡入淡出
- 永久商店：价格公式、解锁树、七项升级效果即时生效
- 随机祝福：每关首入 3（或 4）选 1、按关持久
- 挑战与宝箱：60 秒计时、唯一 runId 计分、奖励先入队后弹窗、每关一次
- 机器人与金币宠物：真实物理箭矢自动射击、宠物寻路复用统一拾取
- 暂停恢复：弹窗、祝福、宝箱、设置与页面隐藏统一暂停
- 本地存档：localStorage 持久化、防抖+关键即时保存、损坏恢复
- UI 与音效：顶部 HUD、底部按钮与冷却条、祝福提示、基础程序化音效与音量设置

## 存档与重置

- 存档写入浏览器 `localStorage`，key 为 `arrowbound_save`；损坏存档会被备份到 `arrowbound_save_corrupt` 并回退到默认存档。
- 存档包含金币、当前关、解锁与通关状态、商店等级、每关祝福、挑战与宝箱领取、待领取奖励、幸运首箭、设置和统计数据。
- 普通金币变化最多 2 秒防抖保存；购买、祝福、通关、挑战、宝箱、奖励和设置变化立即保存；页面关闭前强制保存。
- 游戏内点击底部“设置”，再点“清除存档并重新开始”即可清档重开；也可在浏览器开发者工具中删除 `arrowbound_save`。

## 配置

数值与内容均配置化，集中在 `src/game/config/`：`level.config.ts`（10 关参数）、`shop.config.ts`（商店与解锁树）、`blessing.config.ts`（祝福）、`robot.config.ts`（机器人概率）、`game.constants.ts`（全局常量）、`asset-manifest.ts`（SVG 资源）。调整数值只改配置，不改系统逻辑。

## 核心玩法

```text
选择关卡祝福
→ 把握弓的摆动时机射箭
→ 箭矢按抛物线飞行
→ 根据实际靶面交点判断环数
→ 拾取命中后掉落的金币
→ 购买永久升级
→ 花费目标金币完成关卡并解锁下一关
```

每关还可以主动开启 60 秒挑战。挑战成功后可以获得金币、免费商店升级或额外祝福候选等奖励。

## 首版 MVP

- 1280 × 720 横版 2D 游戏画面
- 玩家固定站位、弓自动摆动、点击射箭
- 真实重力抛物线和连续靶面碰撞检测
- 1～10 环判定、金币掉落和鼠标拾取
- 10 个完整配置关卡
- 永久商店升级和商品解锁树
- 每关首次进入时随机祝福 3 选 1
- 60 秒挑战和一次性挑战宝箱
- 机器人概率瞄准与真实物理射箭
- 金币宠物自动拾取
- 弹窗和浏览器隐藏时暂停
- localStorage 本地存档及异常恢复

## 本地开发

要求 Node.js 22.12 或更高版本。

```bash
npm install
npm run dev
```

完整质量检查：

```bash
npm run check
```

该命令依次执行严格类型检查、单元测试、全仓库 emoji 扫描和生产构建。

涉及场景、画布缩放、素材加载、输入或动效的改动还应运行浏览器验证：

```bash
npm run check:browser
```

该命令自动在 `127.0.0.1:4173` 启动临时 Vite 服务，验证两种桌面视口的画布尺寸、SVG 加载链和浏览器错误，并把结果写入 `/tmp/arrowbound-browser-check/summary.json`。

视觉验收时显式生成画布和射击动效截图：

```bash
npm run check:browser -- --interaction-screenshots
```

截图包括射击标准姿态、释放、冷却双击和复位状态，需要人工查看，不能只根据脚本退出码判断动效观感。

浏览器验证要求 Python 3、Python `playwright` 包及本机 Chrome 或 Chromium。可使用以下环境变量覆盖默认路径：

```bash
ARROWBOUND_CHROME_PATH=/path/to/chrome npm run check:browser
ARROWBOUND_BROWSER_OUTPUT=/tmp/custom-output npm run check:browser
```

验证已经运行的开发服务器：

```bash
python3 scripts/check-browser.py --url http://127.0.0.1:5173/
```

项目包含 Phaser 官方项目级技能，位于 `.agents/skills/`。开发 Phaser 功能时应加载对应技能，并优先采用 Phaser 4.x API。

技能对 Codex 与 Claude Code 双向可用：`.agents/skills/` 为唯一真实来源（Codex 经 `AGENTS.md` 读取），`.claude/skills/` 下每个技能是指向 `../../.agents/skills/<name>` 的符号链接，供 Claude Code 自动发现。新增上游技能时同步补一个符号链接。

技能来源：[phaserjs/phaser/skills](https://github.com/phaserjs/phaser/tree/master/skills)，当前导入版本为上游提交 `539e718ef0c19ddb0777645f2a470eccc8715691`。

首版只适配桌面网页。账号、云存档、排行榜、移动端和 3D 宝箱不属于当前 MVP。

## 关键设计原则

- 射箭手感优先于外围系统
- 射箭收益必须经过金币掉落和拾取流程后入账；宝箱奖励由奖励系统直接发放
- 机器人提供稳定收益，但不能取代玩家操作
- 手动命中 10 环的收益保持为机器人平均单箭收益的约 3～5 倍
- 关卡、商店、祝福、机器人和挑战全部配置化
- 奖励先写入待领取队列，再显示弹窗，避免刷新重复抽奖
- 所有视觉素材绘制为 SVG 文件并放入 `public/assets/svg/`
- 项目和游戏内容不使用 emoji，所有图标由 SVG 素材提供
- 动效必须兼顾即时反馈、信息清晰和重复游玩的节奏
- 开发前优先检查 TODOLIST，每完成一个任务立即更新其状态
