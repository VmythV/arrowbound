# arrowbound 首版 MVP TODOLIST

## 1. 目标

交付一个可在桌面浏览器完整游玩的 arrowbound 首版 MVP，用于验证：

- 抛物线射箭是否有操作乐趣
- 命中、金币掉落和拾取反馈是否清晰
- 商店、关卡和祝福能否形成持续成长动力
- 挑战、机器人和金币宠物能否支撑中期玩法
- 本地存档能否稳定恢复完整进度

首版技术栈固定为 Phaser + TypeScript + Vite，画布为 1280 × 720 并等比缩放。

---

## 2. 已固化规则

- [x] 英文名和工程名统一使用 **arrowbound**
- [x] 每关只有一个普通通关金币目标
- [x] 达到目标后，花费等额金币完成当前关并解锁下一关
- [x] 仅达到目标不自动记录通关
- [x] 挑战金币在拾取时计分
- [x] 只有本次挑战生成、且在倒计时内拾取的金币计入挑战分数
- [x] 商店、祝福、宝箱和设置弹窗打开时暂停游戏与挑战计时
- [x] 十环扩大后，其余 9 个环使用剩余半径等宽分布
- [x] “所有金币”类祝福同时作用于手动和机器人收益
- [x] 宝箱的额外祝福选项奖励需要持久化
- [x] 机器人和金币宠物属于首版 MVP 必做范围

开发前确认项：

- [x] 确认 `05_首版数值配置.md` 中的 C1～C5
- [x] 将确认结果同步回玩法、架构和数值配置文档
- [x] 确认首版数值配置不再包含待定选择或范围值
