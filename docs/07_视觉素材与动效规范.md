# arrowbound 视觉素材与动效规范

## 1. 适用范围

本文档是首版开发的强制规范，适用于游戏画面、UI、素材文件、动效、文档和代码中的用户可见内容。

核心要求：

- 所有视觉素材必须绘制为 SVG 文件，并存放在统一素材资源目录中。
- 项目和游戏内容中不得使用 emoji。
- 所有动效必须先定义用途、节奏、反馈层级和中断规则，再进入实现。
- 动效服务于玩家理解、操作反馈和情绪节奏，不以增加画面复杂度为目标。

---

## 1.5 视觉方向：暖色卡通（已采纳）

首版采用**暖色卡通世界 + 高对比爽感反馈**方向，完整设计稿见 `docs/08_视觉重构稿_暖色卡通.md`，配色/字型/圆角/描边等常量落地在 `src/game/config/theme.ts`，是场景与 UI 引用的唯一来源。要点：

- 造型：统一 4–6px 粗描边、圆角、胖萌比例、两段平涂 + 一道边缘高光；角色吉祥物化。
- 世界层暖而舒适（晨曦天空 + 分层草地），奖励瞬间用高对比色（金 `#FFD24A`、十环品红 `#FF5C8A`）打成全场焦点。
- UI 面板深梅紫 `#241B36` + 暖金描边 `#F4B860`；主行动（下一关/购买）用暖橙 CTA `#FF9F45→#FFC65C`。
- 字型：圆体展示字（Baloo 2 / Fredoka，未打包时回退系统圆体）用于标题与数字，正文 Noto Sans SC。

不要在系统或场景里散布颜色/字体字面量，统一从 `theme.ts` 引用。完整调色板与组件规范见 `docs/08`。

## 1.6 爽感反馈动效（暖色卡通新增）

以下动效服务于「命中即爽」和「目标常在眼前」，均短促、不遮挡箭/靶/金币/按钮/倒计时。

### A. 玩家十环全屏闪

- 触发：**仅玩家手动命中 10 环**（机器人十环不触发，保持手动峰值独占）。
- 表现：主相机一次品红闪 `#FF5C8A`，叠加原有火花放大、白色涟漪、靶心 Q 弹回弹与轻微震屏。
- 时长/缓动：闪光 150ms 线性淡出；不可与自身叠加（新触发前不强制打断已有，交由相机自然淡出）。
- 注意力：全场峰值，机器人命中一律弱化。
- 暂停：暂停时不触发（`paused` 为真时跳过）。
- 验收：手动十环出现明显品红闪 + 火花 + 靶弹；机器人十环无全屏闪；暂停期间不触发。

### B. 达标目标进度条脉冲

- 触发：当前关金币达到通关目标（进度条填满）时。
- 表现：进度条填充转成功绿并做透明度呼吸脉冲（`alpha 1→0.6→1`）循环，直到不再满足（切关/扣费后停止并复位 `alpha=1`）。
- 时长/缓动：单次 620ms，`Sine.InOut`，无限循环。
- 注意力：中，牵引玩家点击「下一关」。
- 验收：达标即脉冲；金币回落到目标以下或通关后停止且进度条不残留半透明。

### C. 下一关 CTA 弹出

- 触发：「下一关」从不可用变为可用（够钱可通关）的那一刻，一次性。
- 表现：按钮 `scale 1.18→1` 弹出，配合暖橙 CTA 底色。
- 时长/缓动：320ms，`Back.Out`；仅在可用状态跳变时触发一次，不循环。
- 验收：够钱瞬间按钮弹一下并变橙；反复刷新金币不重复弹跳。

## 2. SVG 素材规范

### 2.1 目录结构

```text
public/
└── assets/
    ├── svg/
    │   ├── backgrounds/
    │   ├── characters/
    │   ├── bows/
    │   ├── arrows/
    │   ├── targets/
    │   ├── coins/
    │   ├── robots/
    │   ├── pets/
    │   ├── chests/
    │   ├── ui/
    │   └── effects/
    └── audio/
        ├── music/
        └── sfx/
```

视觉素材只能放入 `public/assets/svg/` 对应分类目录。音频不是图像素材，单独放入 `public/assets/audio/`。

### 2.2 必须使用 SVG 的内容

- 角色、弓、箭、靶子
- 金币、机器人、宠物、宝箱
- 场景背景和装饰物
- 商店、祝福、挑战和设置 UI 图标
- 按钮图标、状态图标、鼠标提示图标
- 粒子纹理、闪光、命中特效和轨迹纹理
- 占位素材和正式素材

禁止将 PNG、JPG、WebP、GIF 或 emoji 作为视觉素材提交。第三方视觉素材进入项目前也必须重绘或转换为符合规范的 SVG，并确认使用权。

### 2.3 文件要求

- 文件名使用小写英文和连字符，例如 `coin-bag-large.svg`。
- 每个 SVG 必须包含正确的 `viewBox`，不得依赖固定页面尺寸。
- 不得嵌入位图、外部链接、脚本、外部字体或 base64 数据。
- 文字内容应转换为路径，或由游戏运行时文本组件渲染。
- 同类素材使用一致的画布比例、描边宽度、光源方向和颜色变量。
- 透明留白必须服务于动画或碰撞，不保留无意义空白。
- 提交前清理编辑器元数据、隐藏图层和无效节点。
- SVG 文件必须能够被 Phaser 预加载并按指定尺寸生成纹理。

### 2.4 动画拆分

- 需要独立运动的部件必须拆成独立 SVG，例如角色身体、拉弓手臂、弓、箭袋。
- 运行时通过位置、旋转、缩放、透明度和遮罩完成动画，不在每帧修改 SVG DOM。
- Phaser Graphics 只允许用于碰撞调试、轨迹调试和开发辅助，不得代替最终视觉素材。
- 不允许将大量内联 SVG 字符串散落在 TypeScript 中；素材必须以文件方式加载。

### 2.5 素材清单

建立统一 `asset-manifest.ts`，每项至少包含：

```ts
type VisualAssetDefinition = {
  key: string;
  path: string;
  width: number;
  height: number;
  category: string;
};
```

Scene 和 Entity 只能通过素材 key 使用资源，不直接拼接文件路径。

## 3. 禁止 emoji

- 游戏标题、按钮、弹窗、提示、奖励文本和调试面板不得出现 emoji。
- 文档、代码注释、提交到仓库的示例文本和文件名不得出现 emoji。
- 不得使用 emoji 代替金币、声音、设置、锁定、成功或失败图标。
- 所有图标需求使用项目 SVG 素材解决。
- 每次提交前执行全仓库 emoji 扫描；发现后必须删除或替换为文字、ASCII 标记或 SVG 图标。

普通中文、英文、数字、标点和数学符号不属于 emoji，可以正常使用。

## 4. 动效设计原则

### 4.1 玩家心理目标

- 玩家操作后 100 毫秒内必须看到或听到反馈。
- 高频动作必须短、清楚、不遮挡，不让玩家因重复动画感到拖沓。
- 低频重大事件可以更突出，但不得长时间剥夺控制权。
- 成功反馈强调成就感；失败反馈提供信息，不羞辱、不惩罚玩家。
- 动效不能掩盖箭矢轨迹、靶子、金币位置、挑战计时等关键信息。
- 机器人和宠物的反馈弱于玩家手动十环，避免自动系统抢夺注意力。
- 同时发生多个反馈时按重要级别合并或降级，避免画面噪声。

### 4.2 时长层级

| 类型 | 目标时长 | 是否阻塞操作 |
|---|---:|---|
| 按钮、悬停、数字变化 | 60～180 ms | 否 |
| 射箭、普通命中、拾取 | 100～400 ms | 否 |
| 十环、升级、祝福选择 | 300～800 ms | 否，必要时仅短暂停顿 |
| 通关、挑战结果、宝箱 | 600～1800 ms | 可以暂停，但必须可快速确认 |
| 场景切换 | 200～400 ms | 是 |

同一奖励动画第二次及以后出现时，不得比首次更慢。宝箱动画在关键结果已经安全写入存档后允许玩家加速。

### 4.3 缓动和运动

- UI 打开使用 `Cubic.Out`，关闭使用 `Cubic.In`。
- 掉落和弹跳使用 `Quad.Out` 或物理速度，不使用持续弹性抖动。
- 数值飞向 HUD 使用 `Sine.InOut`。
- 强调缩放最多进行一次放大和一次回弹。
- 高频元素不使用持续旋转、呼吸或闪烁。
- 屏幕震动只用于玩家手动十环和重大挑战成功，位移不超过 4 像素，时长不超过 120 毫秒。
- 禁止高频全屏闪白、快速明暗切换和无法关闭的循环特效。

## 5. 完整动效清单

### 5.1 启动与进场

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 加载进度 | 进度条线性填充，百分比同步更新 | 跟随加载 | 不伪造加载时长 |
| 首次显示场景 | 背景淡入，玩家和靶子分别从两侧小幅进入 | 300 ms | 完成前禁止射箭 |
| 关卡切换 | 旧场景淡出，新场景淡入 | 250 + 250 ms | 中间完成对象回收和配置切换 |
| 祝福选择出现 | 遮罩淡入，卡片按 50 ms 间隔上移出现 | 220 ms | 动画结束前可查看但不可重复选择 |

### 5.2 玩家与射箭

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 弓摆动 | 按真实角度匀速往返 | 持续 | 不额外加入装饰性晃动 |
| 点击射箭 | 弓弦释放、手臂回弹、箭立即生成 | 90～180 ms | 箭生成不得等待动画完成 |
| 冷却 | 冷却条线性恢复，完成时轻微高亮 | 跟随冷却 | 不弹窗、不震屏 |
| 箭矢飞行 | 箭头跟随速度方向，必要时显示短轨迹 | 跟随飞行 | 轨迹不能遮挡下一箭判断 |
| 箭插入靶面 | 箭在实际交点停止并轻微震动一次 | 120 ms | 环数在交点计算后立即显示 |
| 脱靶 | 箭飞出或落地，显示小型尘土和“脱靶”文字 | 250～350 ms | 不使用红屏或长时间失败音效 |

射击动效的实现细则：

- 仅在一次手动射击输入被冷却和状态规则接受后触发；无效输入不播放假反馈。
- 箭矢在接受输入的同一帧生成。弓弦先用 `Cubic.In` 在 55 毫秒内回到弓身，再用 `Cubic.Out` 在 85 毫秒内恢复拉弦外观；拉弓手臂同时用 `Cubic.Out` 在 70 毫秒内回弹 7 度并原路返回，总时长 140 毫秒。
- 射击动效注意力级别为 1，不改变真实弓角、箭速、冷却或输入状态，也不阻塞下一次合法输入。
- 新的合法射击会停止尚未结束的弓弦和手臂动效、重置到标准姿态后重新播放；游戏暂停时动效冻结，恢复后从原进度继续。
- 验收标准：箭矢在输入后 100 毫秒内清晰可见，弓弦和手臂在 180 毫秒内恢复标准姿态，连续射击不产生累积偏移。

命中与脱靶动效的实现细则：

- 命中后箭尖固定在连续检测得到的实际交点，箭身使用 `Sine.InOut` 在 120 毫秒内进行一次不超过 1.5 度的往返震动，随后回收。
- 脱靶反馈在箭矢落地或离开扩展边界时触发；尘土 SVG 使用 `Quad.Out` 轻微放大并淡出，“脱靶”文字上移并淡出，总时长 300 毫秒，注意力级别为 1。
- 新脱靶反馈替换尚未结束的旧脱靶反馈，避免高频叠加；游戏暂停时反馈冻结，场景卸载时立即清理。
- 验收标准：命中箭不会穿过靶面，脱靶反馈不遮挡靶子或 HUD，所有临时对象在动效完成或场景卸载后销毁。

### 5.3 命中层级

| 命中结果 | 动效设计 | 时长 | 注意力级别 |
|---|---|---:|---:|
| 1～3 环 | 小型命中火花，环数文字淡出 | 180 ms | 1 |
| 4～6 环 | 中型火花，靶子位移 1 像素后回正 | 220 ms | 2 |
| 7～9 环 | 较亮火花，环数文字轻微放大 | 300 ms | 3 |
| 手动 10 环 | 中心波纹、靶子单次回弹、最多 4 像素短震、十环文字 | 500～650 ms | 5 |
| 机器人 10 环 | 中心火花和简短环数文字，不震屏 | 280 ms | 3 |

连续命中时，新命中可以刷新文字和粒子，但不得叠加屏幕震动。玩家手动十环始终高于机器人反馈层级。

### 5.4 金币

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 金币生成 | 从命中点沿小抛物线落向地面 | 300～500 ms | 落点不得覆盖主要按钮 |
| 金币落地 | 根据价值进行 1～2 次小弹跳 | 120～260 ms | 高频金币减少弹跳次数 |
| 鼠标拾取 | 金币缩小并飞向金币 HUD，HUD 数字滚动 | 140～220 ms | 入账逻辑在拾取锁定时执行一次 |
| 宠物拾取 | 宠物接近、金币飞向宠物后再给 HUD 反馈 | 180～300 ms | 不延迟实际入账 |
| 高价值金币 | 增加柔和描边或单次光圈 | 最多 600 ms | 不持续闪烁 |

### 5.5 UI 与商店

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 按钮悬停 | 亮度和缩放轻微增加 | 80 ms | 缩放不超过 1.04 |
| 按钮按下 | 缩放至 0.97 后恢复 | 100 ms | 点击逻辑立即执行一次 |
| 弹窗打开 | 遮罩淡入，面板上移并淡入 | 180 ms | 打开后主游戏已经暂停 |
| 弹窗关闭 | 面板小幅下移并淡出 | 140 ms | 完成后恢复游戏 |
| 购买成功 | 商品卡片高亮一次，等级和效果数字更新 | 260 ms | 金币不足时不播放成功反馈 |
| 购买失败 | 价格文字轻微水平移动一次 | 160 ms | 不连续抖动，不播放惩罚性音效 |
| 商品解锁 | 锁形 SVG 淡出，卡片颜色恢复 | 300 ms | 同批解锁按 60 ms 间隔播放 |

### 5.6 祝福

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 卡片出现 | 依次上移和淡入 | 220 ms | 总等待不超过 400 ms |
| 卡片悬停 | 边框高亮，卡片上移 4 像素 | 100 ms | 不遮挡说明文字 |
| 确认选择 | 所选卡片放大，其他卡片淡出 | 280 ms | 先保存再关闭弹窗 |
| 祝福生效 | HUD 祝福图标淡入 | 180 ms | 使用 SVG 图标，不使用 emoji |

### 5.7 挑战

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 挑战开始 | 3、2、1、开始，每段 350 ms | 1400 ms | 倒计时结束后才启动挑战时钟 |
| 挑战计分 | 进度条和数字平滑更新 | 120 ms | 不遮挡射箭区域 |
| 剩余 10 秒 | 计时颜色改变，每秒轻微脉冲一次 | 10 秒 | 不震屏、不全屏闪烁 |
| 挑战成功 | 时间停止，成功横幅进入，随后打开宝箱 | 500 ms | 先保存奖励，再播放动效 |
| 挑战失败 | 时间停止，失败文字淡入后允许立即返回 | 450 ms | 无震屏、无金币损失表现 |

### 5.8 宝箱与奖励

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 宝箱出现 | 缩放和淡入，落地一次 | 300 ms | 奖励已经存在待领取队列 |
| 宝箱开启 | 箱盖旋转、内部光线展开 | 700 ms | 300 ms 后允许点击加速 |
| 奖励展示 | SVG 图标上移，名称和数值依次出现 | 400 ms | 不重新抽取奖励 |
| 奖励确认 | 图标飞向对应 HUD 或商店入口 | 300 ms | 完成后处理队列下一项 |
| 幸运首箭小宝箱 | 使用简化金币箱开启和金币飞向 HUD | 550 ms | 反馈弱于挑战宝箱 |

### 5.9 机器人与宠物

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 机器人瞄准 | 机器人弓朝求解角度旋转 | 180 ms | 不改变真实弹道角度 |
| 机器人射箭 | 弓弦释放并创建真实箭矢 | 140 ms | 多机器人使用初始时间偏移 |
| 机器人命中 | 使用低于玩家的命中层级 | 180～280 ms | 不触发手动十环震屏 |
| 宠物待机 | 低幅度缓慢浮动 | 1600 ms 循环 | 场上有金币时立即中断 |
| 宠物移动 | 朝目标金币移动，朝向随方向改变 | 跟随移动 | 不使用瞬移 |
| 宠物拾取 | 接触金币时单次缩放回弹 | 160 ms | 拾取锁防止重复入账 |

### 5.10 通关与暂停

| 事件 | 动效设计 | 时长 | 规则 |
|---|---|---:|---|
| 达到通关金币 | 下一关按钮由禁用过渡为可用 | 240 ms | 不自动扣费、不自动切关 |
| 确认通关 | 金币数字扣减、解锁路径点亮 | 600 ms | 原子状态更新成功后播放 |
| 暂停 | 游戏对象定格，遮罩和弹窗进入 | 180 ms | 不继续挑战计时 |
| 恢复 | 遮罩退出，游戏从原时间点继续 | 140 ms | 不补算暂停时间 |

## 6. 动效冲突与中断

优先级从高到低：

```text
场景卸载
> 奖励与通关状态
> 暂停弹窗
> 玩家手动十环
> 普通命中和拾取
> 机器人与宠物反馈
> 待机装饰
```

- 高优先级事件可以取消或降级低优先级动效。
- 场景切换时必须停止当前 Scene 的全部 Tween、Timer、Particle 和音效实例。
- 对象进入对象池前必须停止 Tween 并恢复位置、缩放、旋转、透明度和监听器。
- 暂停时 Tween 和游戏时钟一起暂停；UI 弹窗使用独立 UI 时钟。
- 同一对象不得同时运行两个修改相同属性的 Tween。
- 连续拾取金币时合并 HUD 数字动画，不为每枚金币创建长队列。

## 7. 性能与验收

- 目标运行帧率为 60 FPS。
- SVG 在加载时生成 Phaser 纹理，运行时不反复解析文件。
- 高频粒子使用对象池，并设置明确数量上限和寿命。
- 性能压力测试按 5 个机器人、1 个宠物和金币对象池高水位同时运行执行。
- 动效不能改变碰撞结果、金币数值或状态机执行顺序。
- 关闭或加速宝箱动效不得丢失或重复发放奖励。
- 暂停、切关和刷新后不得残留 Tween、Timer 或输入监听。
- 每个 TODO 涉及的新交互都必须同时验收静态状态、播放状态、中断状态和重复触发状态。

动效完成的判定标准：玩家第一次看到时能理解发生了什么，高频重复时不会感到拖延，关键操作期间不会因特效看不清箭、靶子、金币或计时。
